Fun with
jQuery .show()

I said look, not touch

Jquery's .show() effect is more powerful than it would seem on the surface. When we look at the jQuery api for .show() we get this piece of information:

Description: Display the matched elements.

The most common, and obvious use is to reveal, or show, that which was presumably hidden. In other words we are taking an element from display:hidden; to display:block; when the effect is called. This effect is used quite often, for obvious reasons. And the opposite also: .hide() when used together, they make a great team.

The fun really begins when you use the duration property inside the .show() effect. In the demo I cranked up the duration on the effect and you can see in the inspector that all kinds of attributes are being manipulated over time. Most notably height, width, margins, and opacity.

This is great, by just adding a duration we get some awesome and easy animations (e.g. .show(150);). Keep in mind however, when ever you grow an elements height or width, the point of origin from which the element visually springs is the top left (unless it is floating right, or absolutely positioned to the right or bottom.) This makes sense, but what if we want it to spring from the center?

This is my favorite part: If you think to the box model you will remember that the outermost edge of an element is actually the margin. Being creative with this we can define the top and left margin as exactly negative half of the defined height and width. (Make sure to actually define the height and width in the css, or jQuery wonâ€™t know what to grow the object to.) Moving the outer most edge of the element (the margin) into the center of the element means that the animation will appear to spring from the elements center. (to be clear, for an element that is 100px wide and 100px tall, you would define the margin-top and margin-left as -50px.)
